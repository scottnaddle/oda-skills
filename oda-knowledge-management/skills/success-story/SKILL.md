# 성공사례 작성

ODA 사업의 성과를 수혜자 중심의 스토리로 전달하는 성공사례를 작성한다.

## 성공사례의 목적

성공사례는 보고서가 아니다. 사업이 실제 사람의 삶에 어떤 변화를 만들었는지를 감성적이면서도 근거 있게 전달하는 홍보 도구다. 발주처 홍보, 후속사업 제안, 언론 보도, 내부 소통에 활용된다.

## 성공사례 구조

### 1. 헤드라인
- 변화를 한 문장으로 포착
- 예: "디지털 교실이 바꾼 카자흐스탄 시골 학교의 풍경"

### 2. 리드 (도입부)
- 수혜자 소개 (이름, 나이, 직업, 배경 — 동의 하에)
- 변화 전 상황을 생생하게 묘사
- 독자의 공감을 끌어내는 장면

### 3. 도전 (Challenge)
- 수혜자가 직면했던 문제
- 지역/국가 차원의 맥락
- "왜 이 변화가 필요했는가"

### 4. 개입 (Intervention)
- 사업이 제공한 것 (연수, 시스템, 장비, 멘토링)
- 수혜자의 참여 과정
- 핵심 활동을 구체적으로 (추상적 서술 지양)

### 5. 변화 (Transformation)
- 변화 전/후 구체적 비교
- 수혜자 직접 인용 ("이 연수 후에 제 수업이 완전히 달라졌어요")
- 정량적 변화가 있으면 포함 (점수 향상, 취업률 등)
- 사진 (Before/After)

### 6. 파급효과 (Ripple Effect)
- 수혜자 개인을 넘어선 영향
- 동료, 학생, 커뮤니티로의 확산
- 제도적 변화 (해당 시)

### 7. 전망 (Looking Ahead)
- 수혜자의 미래 계획
- 지속가능성 메시지
- 사업의 장기적 가치

### 8. 사업 정보 박스
```
사업명: [사업명]
기간: YYYY~YYYY
발주처: [KOICA/ADB 등]
수행기관: UBION
수원국: [국가]
수혜자: [대상 집단, 규모]
핵심 성과: [1~2줄]
```

## 작성 원칙

- **사람 중심**: 숫자가 아니라 사람의 이야기가 주인공
- **구체적**: "역량이 강화되었다"가 아니라 "매일 LMS로 수업 자료를 올리게 되었다"
- **정직**: 과장하지 않는다. 작은 변화도 진실되게 전달하면 감동
- **시각적**: 사진, 인포그래픽 위치를 표시
- **동의**: 수혜자의 사진/이름 사용에 대한 서면 동의 필수

## 출력 형식

```
# [헤드라인]

[리드 — 수혜자 소개 및 변화 전 장면]

## 도전
[...]

## 변화의 시작
[사업 개입]

## 달라진 일상
[변화, 인용문, 정량 데이터]

## 퍼져나가는 변화
[파급효과]

## 앞으로
[전망]

---
[사업 정보 박스]
```
