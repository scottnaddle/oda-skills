# /draft-proposal

기술제안서 초안의 전체 작성 프로세스를 순서대로 실행하는 워크플로우.

## 실행 흐름

### Step 1: 기술제안서 본문 (write-technical-proposal)

사업 이해, 목표, 방법론, 활동 계획, 인력, 관리, 범분야, 지속가능성의 8섹션을 작성한다.

- RFP 분석 결과(analyze-rfp)가 있으면 이를 기반으로 작성
- 없으면 사용자에게 사업 개요, 발주처, 수원국, 분야를 확인
- 발주처에 맞는 양식과 구조를 적용

사용자에게 초안을 제시하고 피드백을 받은 후 다음 단계로 진행한다.

### Step 2: 로지컬 프레임워크 (write-logical-framework)

제안서 본문의 목표 체계를 PDM(Project Design Matrix)으로 구조화한다.

- 상위목표 → 사업목적 → 산출물 → 활동의 인과관계 정립
- OVI(지표), MoV(검증수단), 가정(Assumptions) 작성
- SMART 지표 검증

### Step 3: 워크플랜 (write-workplan)

활동 계획을 시간축 위에 배치한다.

- WBS(Work Breakdown Structure) 구성
- 간트차트 작성
- 마일스톤 및 성과물 제출 일정 확정
- 현지 사정(우기, 명절, 학기) 반영

### Step 4: 전문가 배치 (expert-roster)

최적의 팀 구성을 설계한다.

- 포지션별 프로파일 정의
- M/M 산정
- 투입 일정표 작성
- CV 작성 가이드 (해당 시)

## 완료 후 산출물

```
# 기술제안서 패키지: [사업명]

## Part 1: 기술제안서 본문
[8섹션 전문]

## Part 2: 로지컬 프레임워크 (PDM)
[매트릭스 + 지표 상세]

## Part 3: 워크플랜
[WBS + 간트차트]

## Part 4: 투입 인력 계획
[조직도 + M/M + 투입 일정]
```

## 완료 후 제안

- 예산서를 작성하려면: "예산서를 작성하시겠습니까? → `/plan-budget`"
- 제안서 품질을 검토하려면: "품질 검토를 하시겠습니까? → `/review-proposal`"
- TOR을 별도 작성하려면: "TOR을 작성하시겠습니까? → write-tor 스킬"
