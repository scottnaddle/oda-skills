# /build-logframe

로지컬 프레임워크를 처음부터 체계적으로 만들거나 기존 프레임워크를 개선하는 워크플로우.

## 실행 흐름

### Step 1: 문제 분석

사업이 해결하려는 핵심 문제를 구조화한다.
- 핵심 문제 정의
- 원인-결과 분석 (Problem Tree)
- 문제를 목표로 전환 (Objective Tree)

사용자에게 문제 구조를 제시하고 확인받는다.

### Step 2: PDM 매트릭스 작성

4×4 매트릭스를 채운다.
- 상위목표, 사업목적, 산출물, 활동 정의
- 각 수준의 OVI(지표) 설정
- MoV(검증수단) 결정
- 수준 간 가정(Assumptions) 도출

### Step 3: 지표 검증

모든 지표에 대해 SMART 기준을 검증한다.
- 구체성, 측정가능성, 달성가능성, 적절성, 시한 확인
- 기초선(Baseline) 확인 필요 여부 표시
- 목표치(Target)의 현실성 검토

## 완료 후 제안

- 제안서 전체를 작성하려면: → `/draft-proposal`
- M&E 체계를 설계하려면: → `/setup-mne` (oda-monitoring-evaluation 플러그인)
