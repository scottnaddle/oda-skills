# /review-proposal

완성된 제안서를 제출 전에 종합 검토하는 워크플로우.

## 실행 흐름

### Step 1: 품질 검토 (proposal-quality-check)

5단계 프레임워크로 제안서를 검토한다.

1. **RFP 충족도**: 배점표 항목, 자격요건, 성과물, 형식 요건 체크
2. **논리적 일관성**: 로지컬 프레임워크-본문-워크플랜-예산 간 정합성
3. **차별화 및 설득력**: UBION 강점 부각, 방법론 구체성, 현장 이해도
4. **가독성**: 구조, 시각화, 문장 명확성, 오탈자
5. **최종 체크리스트**: 표지, 목차, CV, 첨부서류, 제출 형식

### Step 2: 자가 점수 (bid-scoring — oda-bid-intelligence 연계)

RFP 배점표가 있으면 항목별 자가 채점을 실시한다.
- 현재 예상 점수 산출
- 개선 시 기대 점수 추정
- Go/No-Go 최종 판단

## 완료 후 산출물

```
# 제안서 품질 검토 보고서: [사업명]

## 종합 평가
[등급, 핵심 강점, 핵심 개선사항]

## 상세 검토 결과
[5단계 검토 결과]

## 개선 우선순위
[영향도-난이도 매트릭스]

## 자가 점수 (해당 시)
[배점표 기반 채점 결과]
```

## 완료 후 제안

- 특정 섹션을 수정하려면: 해당 스킬 활용 (write-technical-proposal 등)
- 문법/표현을 교정하려면: → `/proofread` (oda-toolkit 플러그인)
