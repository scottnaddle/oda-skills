# 제안서 품질 검토

완성된 (또는 작성 중인) 제안서를 평가위원의 시선으로 검토하여, 제출 전 개선 기회를 식별한다.

## 왜 자체 품질 검토가 필요한가

제안서를 쓴 사람은 자기 글의 약점을 보기 어렵다. 평가위원은 2~3시간 안에 제안서를 읽고 채점하므로, 논리의 비약, 누락된 항목, 모호한 표현이 바로 감점으로 이어진다. 제출 전 한 번의 체계적 검토가 점수를 크게 올릴 수 있다.

## 검토 프레임워크 — 5단계

### 1단계: RFP 충족도 검토 (Compliance Check)

RFP의 모든 요구사항이 제안서에 반영되어 있는지 확인한다:

- [ ] 배점표의 모든 항목이 제안서에서 다뤄지고 있는가
- [ ] 필수 자격요건이 증빙과 함께 제시되어 있는가
- [ ] 요구된 성과물 목록이 빠짐없이 포함되어 있는가
- [ ] 페이지 수, 글자 크기, 양식 등 형식 요건을 준수하는가
- [ ] 필수 첨부 서류가 모두 준비되어 있는가
- [ ] 범분야 이슈(젠더, 환경 등)가 반영되어 있는가

### 2단계: 논리적 일관성 검토 (Logic Check)

제안서 내부의 논리적 일관성을 검증한다:

- [ ] 로지컬 프레임워크의 목표 체계와 본문 서술이 일치하는가
- [ ] 산출물→활동→방법론 간 인과관계가 명확한가
- [ ] 워크플랜의 활동이 본문의 활동과 1:1 대응하는가
- [ ] 투입 인력의 M/M이 워크플랜과 일치하는가
- [ ] 예산서의 비목이 활동 계획과 연동되는가
- [ ] 숫자(기간, 인원, 예산)가 문서 간 일치하는가

### 3단계: 차별화 및 설득력 검토 (Impact Check)

평가위원에게 높은 인상을 줄 수 있는지 검토한다:

- [ ] UBION만의 차별화 포인트가 명확하게 드러나는가
- [ ] 방법론이 구체적이고 혁신적인가 (추상적 원론이 아닌)
- [ ] 수원국 현장에 대한 깊은 이해가 보이는가
- [ ] 한국 경험/기술의 적용이 자연스러운가
- [ ] 지속가능성 방안이 현실적이고 구체적인가
- [ ] 리스크를 사전에 인식하고 대응 방안을 제시하는가

### 4단계: 가독성 및 표현 검토 (Readability Check)

평가위원이 읽기 쉬운지 검토한다:

- [ ] 각 섹션의 첫 단락에 핵심 메시지가 있는가
- [ ] 표, 그림, 다이어그램으로 시각적 명료성을 높였는가
- [ ] 문장이 간결하고 명확한가 (한 문장에 하나의 메시지)
- [ ] 전문 용어가 과도하지 않은가
- [ ] 오탈자, 문법 오류가 없는가
- [ ] 번호, 참조, 약어가 일관적인가

### 5단계: 최종 체크리스트

제출 직전 확인:

- [ ] 제안서 표지에 사업명, 제출자, 날짜가 정확한가
- [ ] 목차의 페이지 번호가 정확한가
- [ ] 모든 표와 그림에 번호와 제목이 있는가
- [ ] CV가 최신 버전이고 서명이 있는가
- [ ] 전자 입찰 시스템의 파일 형식/용량 제한을 충족하는가
- [ ] 제출 마감일시를 최종 확인했는가

## 출력 형식

```
# 제안서 품질 검토 보고서: [사업명]

## 1. 종합 평가
- 전체 품질 등급: [A/B/C/D]
- 핵심 강점 3가지
- 핵심 개선 필요사항 3가지

## 2. RFP 충족도
[체크리스트 결과 + 미충족 항목 상세]

## 3. 논리적 일관성
[불일치 항목 + 수정 제안]

## 4. 차별화 및 설득력
[강화 필요 영역 + 구체적 제안]

## 5. 가독성 및 표현
[수정 필요 문장/표현 목록]

## 6. 개선 우선순위
| 순위 | 항목 | 영향도 | 수정 난이도 | 권고 |
|---|---|---|---|---|
| 1 | [항목] | 높음 | 쉬움 | 즉시 수정 |
| 2 | ... | ... | ... | ... |
```
