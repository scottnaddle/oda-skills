# 워크플랜 수립

ODA 사업의 전체 실행 일정을 체계적으로 설계한다. 로지컬 프레임워크의 활동을 시간축 위에 배치하고, 인력 투입과 성과물 제출 일정을 통합한다.

## 작성 절차

### 1단계: WBS (Work Breakdown Structure)

로지컬 프레임워크의 산출물-활동 구조를 분해한다:

```
산출물 1: [명칭]
├── 활동 1.1: [명칭]
│   ├── 세부과업 1.1.1
│   └── 세부과업 1.1.2
├── 활동 1.2: [명칭]
└── 활동 1.3: [명칭]

산출물 2: [명칭]
├── 활동 2.1: [명칭]
...
```

### 2단계: 활동 일정 배치

각 활동의 시작일, 종료일, 기간을 결정한다:

- **선후관계(Dependencies)**: 어떤 활동이 먼저 완료되어야 다음 활동이 가능한가
- **병행 가능성**: 동시에 수행할 수 있는 활동은 병렬 배치
- **현지 사정 반영**: 우기, 명절(라마단, 설, 추석), 선거 기간, 학기 일정
- **마일스톤**: 핵심 성과물 제출, PSC 회의, 중간평가 시점

### 3단계: 인력 투입 일정

전문가별 투입 시기를 워크플랜과 연동한다:

```
| 전문가 | 1Q | 2Q | 3Q | 4Q | 5Q | 6Q | 합계(M/M) |
|---|---|---|---|---|---|---|---|
| PM | ███ | ███ | ███ | ███ | ███ | ███ | 12 |
| 교육전문가 | ░░░ | ███ | ███ | ███ | ░░░ | ░░░ | 6 |
| ICT전문가 | ░░░ | ░░░ | ███ | ███ | ███ | ░░░ | 6 |

███: 풀타임, ░░░: 파트타임/대기
```

### 4단계: 간트차트 구성

활동, 마일스톤, 성과물을 간트차트로 통합한다:

```
| 활동 | M1 | M2 | M3 | M4 | M5 | M6 | ... | M12 |
|---|---|---|---|---|---|---|---|---|
| 1.1 착수 보고 | ██ | | | | | | | |
| 1.2 기초선 조사 | | ██ | ██ | | | | | |
| 2.1 커리큘럼 개발 | | | ██ | ██ | ██ | | | |
| 2.2 교원 연수 | | | | | ██ | ██ | ██ | |
| 3.1 시스템 구축 | | ██ | ██ | ██ | ██ | | | |
| ◆ 중간보고 | | | | | | ◆ | | |
| ◆ 종료보고 | | | | | | | | ◆ |
```

### 5단계: 성과물 제출 일정

```
| 성과물 | 제출 시기 | 검수 기간 | 비고 |
|---|---|---|---|
| 착수보고서 | M1 | 2주 | PSC 승인 |
| 기초선 조사보고서 | M3 | 2주 | |
| 중간보고서 | M6 | 2주 | 중간평가 연계 |
| 최종보고서 | M12 | 4주 | 종료평가 포함 |
```

## 작성 원칙

- 워크플랜은 로지컬 프레임워크, 예산서, 인력 투입표와 완벽히 일치해야 한다
- 현실적인 일정을 제시한다 — 여유 없는 일정은 리스크를 높인다
- 수원국 카운터파트의 역할과 일정도 포함한다
- Critical Path를 식별하여 일정 리스크를 관리한다
