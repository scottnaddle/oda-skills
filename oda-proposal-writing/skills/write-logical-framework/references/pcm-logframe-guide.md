# PCM 로지컬 프레임워크 상세 가이드

## PCM (Project Cycle Management) 개요

PCM은 개발 프로젝트의 기획, 실행, 평가를 체계적으로 관리하기 위한 방법론이다. KOICA, JICA, EU 등 주요 공여기관이 채택하고 있다.

### PCM 5단계
1. **기획(Formulation)**: 문제 분석, 목표 분석, 프로젝트 설계
2. **심사(Appraisal)**: 타당성 검토, 사전 평가
3. **실행(Implementation)**: 사업 수행, 모니터링
4. **종료(Completion)**: 사업 종료, 종료 평가
5. **사후 평가(Ex-post Evaluation)**: 영향 평가, 교훈 도출

## PDM (Project Design Matrix) 상세

### 표준 PDM 양식

```
┌──────────┬────────────────────┬──────────────────┬──────────────────┬──────────────────┐
│ 구분      │ 프로젝트 요약       │ 지표 (OVI)       │ 입수수단 (MoV)   │ 외부조건/가정     │
├──────────┼────────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ 상위목표  │ [사업이 기여하는     │ [상위목표 수준    │ [데이터 소스]     │                  │
│ (Goal)   │  장기적 목표]        │  지표]           │                  │                  │
├──────────┼────────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ 사업목적  │ [사업이 직접        │ [사업목적 수준    │ [데이터 소스]     │ [사업목적→상위    │
│(Purpose) │  달성하는 변화]      │  지표]           │                  │  목표 가정]       │
├──────────┼────────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ 산출물    │ [사업 활동의        │ [산출물 수준      │ [데이터 소스]     │ [산출물→사업     │
│(Outputs) │  구체적 결과물]      │  지표]           │                  │  목적 가정]       │
├──────────┼────────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ 활동      │ [산출물을 생산하기   │                  │                  │ [활동→산출물     │
│(Activities)│ 위한 구체적 활동]   │  투입 (Inputs)   │  예산            │  가정/전제조건]   │
└──────────┴────────────────────┴──────────────────┴──────────────────┴──────────────────┘
```

### 지표 유형별 예시

**상위목표 수준 (Impact)**
- 국가 교육 통계 변화 (취학률, 문해율, 졸업률)
- HDI 교육 지수 변화
- 노동시장 진입률 변화

**사업목적 수준 (Outcome)**
- 대상 기관/지역의 교육 성과 변화
- 수혜자의 역량/행동 변화 (사전-사후 비교)
- 시스템/제도의 기능 변화

**산출물 수준 (Output)**
- 개발/제공된 교육 프로그램 수
- 연수/워크숍 완료 인원
- 구축된 시스템/플랫폼
- 개발된 교재/콘텐츠 수

### 가정(Assumptions) 예시

**활동→산출물 수준**
- 수원국 카운터파트가 적시에 참여자를 선발한다
- 현지 인터넷 인프라가 시스템 운영에 충분하다
- 전문가 비자 발급이 원활히 이루어진다

**산출물→사업목적 수준**
- 연수 참여자가 소속 기관에서 학습 내용을 적용한다
- 수원국 정부가 개발된 커리큘럼을 공식 채택한다
- 구축된 시스템을 지속적으로 유지관리한다

**사업목적→상위목표 수준**
- 수원국 정부의 교육 정책이 일관되게 유지된다
- 충분한 예산이 지속적으로 배정된다
- 정치적 안정이 유지된다

## KOICA vs ADB 로지컬 프레임워크 차이

| 항목 | KOICA (PDM) | ADB (DMF) |
|---|---|---|
| 명칭 | Project Design Matrix | Design and Monitoring Framework |
| 최상위 | 상위목표 (Goal) | Impact |
| 2단계 | 사업목적 (Purpose) | Outcome |
| 3단계 | 산출물 (Outputs) | Outputs |
| 4단계 | 활동 (Activities) | Activities with Milestones |
| 지표 | OVI | Performance Indicators |
| 기초선 | 선택 | 필수 (Baselines and Targets 별도 열) |
| 리스크 | 가정 (Assumptions) | Risks |

### ADB DMF 추가 요소
- **Baselines**: 각 지표의 현재 값 (사업 시작 전)
- **Targets**: 연도별 목표치 (중간, 종료)
- **Data Sources and Methods**: 데이터 수집 방법 상세
- **Risks**: 가정보다 구체적인 리스크 기술 + 완화 조치

## 교육 분야 로지컬 프레임워크 예시

### 사업명: 디지털 교육 역량강화 사업 (예시)

| 구분 | 서술 | OVI | MoV | 가정 |
|---|---|---|---|---|
| 상위목표 | 수원국 교육의 질적 수준 향상에 기여 | 대상 지역 학생 학업성취도 10% 향상 (2027) | 국가 학업성취도 평가 결과 | - |
| 사업목적 | 대상 학교 교원의 ICT 활용 교수 역량 강화 | ICT 활용 수업 실시 교원 비율 30%→70% (사업종료시) | 수업 관찰 보고서, 교원 설문조사 | 교원이 학습 내용을 실제 수업에 적용한다 |
| 산출물 1 | ICT 활용 교수법 연수 프로그램 개발 및 운영 | 연수 이수 교원 200명, 만족도 4.0/5.0 이상 | 연수 수료증, 만족도 조사 | 카운터파트가 적시에 연수 참여자를 선발한다 |
| 산출물 2 | 디지털 교육 콘텐츠 및 LMS 플랫폼 구축 | LMS 구축 1식, 교육 콘텐츠 50개 모듈 | LMS 운영 로그, 콘텐츠 목록 | 학교 인터넷 인프라가 LMS 접속에 충분하다 |
| 산출물 3 | 교육 정책 자문 및 가이드라인 개발 | ICT 교육 가이드라인 1건 교육부 승인 | 교육부 공식 문서 | 교육부가 정책 개선에 적극적이다 |
