# TOR (Terms of Reference) 작성

ODA 사업의 TOR 또는 개별 전문가 TOR을 체계적으로 작성한다.

## TOR의 두 가지 유형

1. **사업 TOR**: 전체 사업의 과업 범위를 정의 (발주처가 RFP에 포함하거나, PMC가 시행기관에 제시)
2. **전문가 TOR**: 개별 전문가/컨설턴트의 업무 범위를 정의

## 사업 TOR 표준 구조

```
1. 배경 (Background)
   - 사업의 맥락, 수원국 현황, 필요성

2. 목적 (Objectives)
   - 사업의 전체 목적
   - 구체적 목적 (Specific Objectives)

3. 업무 범위 (Scope of Work)
   - 사업의 전체 범위
   - 포함/제외 사항 명시

4. 세부 업무 내용 (Tasks/Activities)
   - 산출물별 세부 과업
   - 방법론 요구사항 (있는 경우)

5. 성과물 (Deliverables)
   - 필수 성과물 목록
   - 제출 시기 및 양식
   - 검수 기준

6. 투입 인력 (Team Composition)
   - 필요 전문가 구성
   - 각 포지션의 자격요건

7. 사업 기간 및 일정 (Duration & Timeline)
   - 전체 기간
   - 주요 마일스톤

8. 보고 체계 (Reporting)
   - 보고 대상 및 주기
   - 보고서 양식

9. 수원국 카운터파트 역할 (Counterpart Arrangements)

10. 예산 (Budget) — 해당 시
```

## 전문가 TOR 표준 구조

```
1. 포지션명 (Position Title)
2. 배경 (Background)
3. 목적 (Objective)
4. 구체적 업무 (Specific Tasks)
   - 업무 1: [내용, 기대 결과]
   - 업무 2: ...
5. 성과물 (Deliverables)
6. 자격요건 (Qualifications)
   - 학력
   - 경력 (연수, 분야)
   - 언어 능력
   - 기타 (자격증, 국가 경험 등)
7. 투입 기간 (Duration)
8. 근무지 (Duty Station)
9. 보고 대상 (Reports to)
```

상세 양식 가이드 → `references/tor-template.md`

## 작성 원칙

- **명확성**: 모호한 표현을 피한다. "적절히 수행한다" 대신 "월 1회 진도보고서를 제출한다"
- **측정가능성**: 성과물의 완료 기준이 명확해야 한다
- **현실성**: 일정과 범위가 실현 가능한 수준이어야 한다
- **완결성**: 빠진 항목이 없어야 한다. 특히 성과물, 자격요건, 일정은 필수
