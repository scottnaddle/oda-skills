# /manage-risks

리스크 등록부를 생성하거나 기존 등록부를 업데이트하는 워크플로우.

## 실행 흐름

risk-register 스킬을 단독으로 실행한다.

### 신규 생성 모드
사업 시작 시 또는 리스크 등록부가 없을 때:
1. ODA 사업 일반 리스크 카테고리 기반으로 식별
2. 사업 특수 리스크 추가 (사용자 인터뷰)
3. 5×5 매트릭스로 전체 평가
4. 대응 전략 및 담당자 지정
5. 모니터링 주기 설정

### 업데이트 모드
기존 등록부가 있을 때:
1. 기존 리스크 상태 업데이트 (Active/대응중/해소/발현)
2. 신규 리스크 추가 (새로운 이슈 발생 시)
3. 발생확률/영향도 재평가
4. 대응 조치 효과 검토
5. 진도보고서 연계 포인트 정리

## 완료 후 제안

- 진도보고에 반영하려면: → `/write-progress`
- 착수보고에 포함하려면: → `/write-inception`
