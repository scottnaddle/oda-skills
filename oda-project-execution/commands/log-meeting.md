# /log-meeting

회의 내용을 체계적인 회의록으로 변환하는 워크플로우.

## 실행 흐름

meeting-minutes 스킬을 단독으로 실행한다.

### 전사본이 제공된 경우
1. 발언자를 식별하고 분류
2. 안건별로 논의 내용 정리
3. 결정사항 추출
4. 후속조치(Action Items) 추출 — 담당자/기한 포함
5. 다국어 발언은 주 언어로 통합

### 수동 입력의 경우
1. 기본 정보 확인 (일시, 장소, 참석자)
2. 안건별 논의 내용 구조화
3. 결정사항 정리
4. 후속조치 표 작성

## 완료 후 제안

- 이전 회의의 Action Items 상태를 추적하려면: 이전 회의록을 함께 제공
- PSC 회의록이라면: 양측 서명용 양식으로 포맷팅
