# 회의록 작성

ODA 사업 수행 중 발생하는 다양한 회의의 기록을 체계적으로 작성한다.

## 회의록 표준 구조

```
# 회의록: [회의 제목]

## 기본 정보
- 일시: YYYY.MM.DD (요일) HH:MM~HH:MM
- 장소: [장소명 / 화상회의 플랫폼]
- 참석자:
  - [소속] [이름] ([역할])
  - [소속] [이름] ([역할])
- 불참: [이름] (사유)
- 작성자: [이름]

## 안건

### 안건 1: [제목]
**논의 내용:**
- [핵심 발언/논점 요약]
- [반대 의견 또는 대안이 있었다면 포함]

**결정사항:**
- [구체적 결정 내용]

### 안건 2: [제목]
...

## 후속조치 (Action Items)

| # | 조치사항 | 담당자 | 기한 | 비고 |
|---|---|---|---|---|
| 1 | [구체적 액션] | [이름/팀] | MM/DD | |
| 2 | [구체적 액션] | [이름/팀] | MM/DD | |

## 기타/공유 사항
[공유된 정보, 다음 회의 일정 등]

## 다음 회의
- 일시: YYYY.MM.DD
- 안건 (예정): [...]
```

## 회의 유형별 특화

**PSC (운영위원회) 회의**
- 공식 안건 번호 부여
- 승인/의결 사항 명확히 기록 (찬반 표시)
- 양측(발주처/수행기관) 서명란

**카운터파트 실무 회의**
- 한영 이중 기록 (필요 시)
- 합의사항(Agreed Minutes) 형식 활용

**내부 팀 회의**
- 간략한 형식 가능
- 진도 현황 + 이슈 + 다음 주 계획 중심

## 전사본(Transcript)에서 회의록 추출

사용자가 회의 전사본을 제공하면:
1. 발언자를 식별하고 분류한다
2. 안건별로 논의 내용을 정리한다
3. 결정사항과 후속조치를 추출한다
4. 중복/잡담을 제거하고 핵심만 남긴다
5. 다국어 발언은 주 언어로 통합 정리한다

## 다음 단계 제안

- **후속조치를 추적하려면** → 다음 회의 전에 이전 Action Items 상태 확인
- **PSC 회의를 준비하려면** → psc-preparation 스킬 (oda-partnership 플러그인)
