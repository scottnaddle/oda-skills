# 조달계획 수립

ODA 사업 수행 중 필요한 기자재, 소프트웨어, 서비스의 조달을 체계적으로 계획한다.

## 조달계획 표준 구조

### 1. 조달 대상 목록

```
| # | 품목 | 유형 | 수량 | 예상금액 | 조달방법 | 시기 |
|---|---|---|---|---|---|---|
| 1 | 노트북 | HW | 20대 | $15,000 | 경쟁입찰 | M3 |
| 2 | LMS 라이선스 | SW | 1식 | $8,000 | 수의계약 | M4 |
| 3 | 서버 호스팅 | 서비스 | 12개월 | $6,000 | 수의계약 | M2 |
```

### 2. 조달 방법 결정

발주처 규정에 따라:
- **경쟁입찰**: 일정 금액 이상 (KOICA: 보통 2,000만원↑, ADB: $100K↑)
- **수의계약/쇼핑**: 소액 또는 특수 품목
- **직접구매**: 현지 소액 구매

### 3. 사양서 (Specification)

각 품목에 대해:
- 기술 사양 (최소/권장)
- 수량 및 단위
- 납품 일정 및 장소
- 보증/유지보수 조건
- 소유권 이전 조건 (사업 종료 시 수원국 이전 여부)

### 4. 조달 일정

```
| 단계 | 기간 | 비고 |
|---|---|---|
| 사양서 확정 | 2주 | 카운터파트 협의 |
| 입찰 공고 | 2~4주 | 발주처 승인 후 |
| 평가 및 선정 | 1~2주 | |
| 계약 및 납품 | 2~6주 | 품목에 따라 |
| 검수 | 1주 | |
```

### 5. 유의사항

- 발주처 사전 승인이 필요한 품목 확인 (ADB: Prior Review 대상)
- 현지 조달 vs 한국 조달 vs 제3국 조달 결정
- 관세/통관 절차 확인 (ODA 면세 적용 여부)
- ICT 장비의 현지 호환성 (전압, 플러그, 인터넷 규격)
- 소프트웨어 라이선스의 현지 사용 제한 확인
