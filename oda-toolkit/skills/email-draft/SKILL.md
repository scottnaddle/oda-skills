# ODA 업무 이메일 작성

ODA 사업 수행 중 발생하는 다양한 업무 이메일을 상황과 수신자에 맞게 작성한다.

## 이메일 유형별 가이드

### 1. 발주처(KOICA/ADB) 보고/요청
- 톤: 격식, 정중, 간결
- 구조: 인사 → 목적 → 핵심 내용 → 요청사항 → 마무리
- 첨부: 보고서, 증빙 등 명확히 안내
- 주의: 결재 라인 확인, CC 범위 확인

### 2. 카운터파트(수원국 정부) 소통
- 톤: 공식적이되 협력적
- 언어: 영어 (현지어 인사말 포함 시 호감)
- 구조: 감사/인사 → 미팅 후속 → 합의사항 확인 → 다음 단계
- 주의: 문화적 감수성, 직위 정확히 표기

### 3. 전문가 섭외/조율
- 톤: 전문적이되 친근
- 구조: 사업 소개 → 역할 설명 → 일정/조건 → 회신 요청
- 포함: TOR 요약, 투입 기간, 보수 수준 (또는 협의)

### 4. 컨소시엄 파트너 소통
- 톤: 동료적, 실무적
- 구조: 현안 → 각자 역할 → 기한 → 확인 요청
- 주의: 민감한 사안(예산, 지분)은 대면 후 이메일로 확인

### 5. 미팅 요청/확인
- 톤: 간결, 명확
- 포함: 일시, 장소, 참석자, 안건
- 구조: 목적 1줄 → 일시/장소 → 안건 → 확인 요청

## 이메일 템플릿

### 영문 공식 서한 기본 구조

```
Subject: [사업명] — [구체적 목적]

Dear [Title] [Last Name],

[Opening: 인사 또는 감사]

[Body: 핵심 내용 — 1~3 단락]

[Closing: 요청사항 또는 다음 단계]

Best regards,
[Name]
[Title], [Organization]
[Phone] | [Email]
```

### 한글 공식 서한 기본 구조

```
제목: [사업명] [건명]

[수신자 직위/성함] 귀하,

[인사]

[본문: 핵심 내용]

[요청사항 또는 안내]

[마무리 인사]

[소속/직위/성명]
[연락처]
```

## 작성 원칙

- 제목(Subject)에 사업명과 핵심 용건을 반드시 포함
- 핵심 메시지를 첫 단락에 배치 (바쁜 수신자가 첫 줄만 읽을 수 있음)
- 요청사항이 있으면 기한을 명시
- 첨부 파일이 있으면 본문에서 언급
- 참조(CC) 대상을 신중히 결정
