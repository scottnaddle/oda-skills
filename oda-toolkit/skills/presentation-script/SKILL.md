# 발표 대본 작성

ODA 사업 관련 프레젠테이션의 발표 대본과 질의응답 준비 자료를 작성한다.

## 발표 유형별 가이드

### 제안서 발표 (Proposal Presentation)
- 목적: 평가위원에게 제안서의 핵심을 설득
- 시간: 보통 20~30분 발표 + 10~20분 Q&A
- 핵심: 차별화 포인트 강조, 자신감 있는 톤
- 발표자: PM (RFP에서 지정하는 경우 많음)

### PSC/운영위 발표
- 목적: 사업 진행 현황 보고, 이슈 협의, 승인 획득
- 시간: 30~60분
- 핵심: 성과 중심, 이슈에 대한 명확한 대응 방안
- 청중: 발주처 + 카운터파트 고위급

### 성과 공유 세미나
- 목적: 사업 성과 홍보, 교훈 공유
- 시간: 15~20분
- 핵심: 스토리텔링, 수혜자 변화, 시각적 증거
- 청중: 다양 (정부, 학계, 언론, 일반)

## 대본 작성 원칙

### 통역 친화적 문장 구조
통역이 있는 발표(대부분의 ODA 발표)에서:
- 한 문장에 하나의 메시지만 담는다
- 문장을 짧게 끊는다 (15~20단어 이내)
- 숫자, 고유명사는 천천히 말한다
- 전문 용어 사용 시 첫 등장에서 풀어서 설명
- 슬라이드 전환 시 자연스러운 연결어

### 슬라이드별 대본 구조

```
---
슬라이드 3: [슬라이드 제목]
[슬라이드 내용 요약]

대본:
"[발표 내용 — 자연스러운 구어체]"

포인트:
- [이 슬라이드에서 강조할 핵심 1~2가지]

전환:
"[다음 슬라이드로 넘어가는 연결 멘트]"
---
```

## Q&A 준비

### 예상 질문 카테고리
- **방법론**: "이 접근법이 현지에서 실제로 작동하는가?"
- **인력**: "PM의 해당 국가 경험은?"
- **일정**: "이 일정이 현실적인가?"
- **예산**: "이 단가의 근거는?"
- **지속가능성**: "사업 종료 후 어떻게 유지되는가?"
- **리스크**: "가장 큰 리스크는 무엇이고 어떻게 대응하는가?"
- **차별화**: "다른 업체와 뭐가 다른가?"

### 답변 구조 (STAR)
- **S**ituation: 질문의 맥락 인정
- **T**ask: 우리가 해결하려는 것
- **A**ction: 구체적 대응 방안
- **R**esult: 기대 효과

## 출력 형식

```
# 발표 대본: [발표 제목]
## 발표 정보
- 일시/장소/청중/시간

## 슬라이드별 대본
[슬라이드 1~N]

## 예상 Q&A
| # | 예상 질문 | 답변 요지 |
|---|---|---|

## 발표 팁
[특별 유의사항]
```
