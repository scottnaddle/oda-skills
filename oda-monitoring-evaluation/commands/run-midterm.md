# /run-midterm

사업 중간 시점의 평가를 설계하고 실행하는 워크플로우.

## 실행 흐름

### Step 1: 중간평가 설계 (midterm-evaluation)

평가 TOR, 평가 매트릭스, 데이터 수집 계획을 수립한다.
- DAC 기준별 평가 질문 설정
- 평가 매트릭스 작성
- 데이터 수집 방법 결정

### Step 2: 데이터 품질 점검 (data-quality-check)

기존 M&E 데이터의 품질을 점검한다.
- 진도보고서 데이터 검증
- 설문 데이터 완전성/정확성 점검
- 활용 가능한 데이터와 추가 수집 필요 데이터 식별

### Step 3: 설문 분석 (survey-analysis)

설문 데이터가 있으면 분석을 실시한다.
- 기술통계 및 교차분석
- 만족도 분석 (해당 시)
- 사전-사후 비교 (해당 시)
- 자유응답 분석

## 완료 후 산출물

중간평가 보고서 (발견-결론-제언 구조)

## 완료 후 제안

- 진도보고에 반영하려면: → `/write-progress` (oda-project-execution 플러그인)
- 리스크를 업데이트하려면: → `/manage-risks` (oda-project-execution 플러그인)
