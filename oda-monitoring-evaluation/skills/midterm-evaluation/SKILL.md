# 중간평가 설계 및 보고

사업 중간 시점에 실시하는 평가를 설계하고, 평가 결과를 체계적으로 보고한다.

## 중간평가의 목적

중간평가는 "채점"이 아니라 "방향 교정"이다. 사업이 올바른 방향으로 가고 있는지 확인하고, 남은 기간에 무엇을 조정해야 하는지 결정하기 위한 형성적(Formative) 평가다.

## 중간평가 설계

### 평가 TOR 구조

```
1. 배경 및 목적
   - 사업 개요
   - 평가 목적 (형성적 평가임을 명시)
   - 평가 범위 (기간, 지역, 산출물)

2. 평가 질문
   - 적절성: 사업 설계가 여전히 타당한가?
   - 효과성: 산출물이 계획대로 달성되고 있는가?
   - 효율성: 자원이 효율적으로 활용되고 있는가?
   - 지속가능성 전망: 사업 종료 후 지속 가능성은 어떠한가?

3. 평가 방법론
   - 데이터 수집 방법 (문헌 검토, 인터뷰, 설문, 현장 방문)
   - 표본 설계
   - 분석 프레임워크

4. 평가팀 구성
5. 일정
6. 성과물 (평가 보고서, 발표)
7. 예산
```

### 평가 매트릭스

각 평가 질문에 대한 데이터 수집 계획:

```
| DAC 기준 | 평가 질문 | 판단 기준 | 데이터 소스 | 수집 방법 |
|---|---|---|---|---|
| 적절성 | 수원국 니즈가 변화했는가? | 니즈 대비 사업 방향 일치도 | 이해관계자 | 인터뷰 |
| 효과성 | 산출물 달성률은? | PDM 지표 목표 대비 실적 | 진도보고서, 설문 | 문헌검토, 설문 |
| 효율성 | 예산 집행이 적정한가? | 집행률, 비용 효과성 | 재무보고서 | 문헌검토 |
```

## 중간평가 보고서 구조

```
# 중간평가 보고서: [사업명]

## Executive Summary (2~3쪽)
[핵심 발견, 결론, 제언 요약]

## 1. 평가 개요
- 평가 목적 및 범위
- 평가 방법론
- 제약사항 (Limitations)

## 2. 사업 현황
- 추진 경과
- 성과 달성 현황 (PDM 기반)

## 3. 발견사항 (Findings)
### 3.1 적절성
### 3.2 효과성
### 3.3 효율성
### 3.4 지속가능성 전망

## 4. 결론 (Conclusions)
[발견사항에서 도출된 판단]

## 5. 제언 (Recommendations)
| # | 제언 | 대상 | 우선순위 | 시기 |
|---|---|---|---|---|
| 1 | [제언] | [수행기관/발주처/카운터파트] | 높음 | 즉시 |

## 6. 교훈 (Lessons Learned)
[향후 사업에 적용 가능한 교훈]

## 별첨
- 평가 TOR
- 면담자 목록
- 설문지
- 데이터 분석 결과
```

## 작성 팁

- 발견사항과 제언은 반드시 근거(evidence)에 기반해야 한다
- 제언은 구체적이고 실행 가능해야 한다. "더 노력해야 한다" 같은 추상적 제언은 무용
- 긍정적 발견도 포함한다. 잘 되고 있는 부분을 인정해야 균형 잡힌 평가가 된다
- 제언에는 반드시 "누구에게" "언제까지" "무엇을" 하라는 것인지 명시
