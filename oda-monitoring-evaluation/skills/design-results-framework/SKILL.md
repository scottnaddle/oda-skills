# 성과 프레임워크 설계

ODA 사업의 전체 성과관리 체계를 설계한다. 사업이 의도하는 변화의 논리(Theory of Change)를 구조화하고, 이를 측정·관리하기 위한 프레임워크를 수립한다.

## 왜 성과 프레임워크가 중요한가

"측정할 수 없으면 관리할 수 없다." ODA 사업에서 성과 프레임워크는 제안서의 약속을 실행 중에 추적하고, 종료 시 달성 여부를 객관적으로 입증하는 유일한 도구다. 발주처(KOICA, ADB 등)는 성과 기반 관리(Results-Based Management)를 점점 더 강조하고 있으며, 잘 설계된 프레임워크는 사업의 신뢰도를 높인다.

## 설계 절차

### 1단계: 변화이론(Theory of Change) 수립

사업이 의도하는 변화의 인과관계를 서술형으로 정리한다:

"**만약** [투입]을 제공하고 [활동]을 수행**하면**, [산출물]이 생산되고, **그 결과** [대상 집단]의 [행동/역량/상태]가 변화하여(성과), **궁극적으로** [상위 목표]에 기여**한다**. 이 인과관계가 작동하려면 [가정 1], [가정 2]가 충족되어야 한다."

변화이론은 로지컬 프레임워크의 상위 개념이다. PDM이 "무엇을, 어떻게 측정하는가"를 다룬다면, 변화이론은 "왜 이 개입이 변화를 만드는가"를 설명한다.

### 2단계: 성과 체인(Results Chain) 구성

```
투입(Inputs)
  ↓ [자원 투입]
활동(Activities)
  ↓ [실행]
산출(Outputs)
  ↓ [직접 결과]    ← 사업 통제 범위
─────────────────────────────────
성과(Outcomes)     ← 사업 영향 범위
  ↓ [행동 변화]
영향(Impact)       ← 장기적 기여
```

각 수준 간 전환에는 **가정(Assumptions)**과 **리스크(Risks)**가 존재한다. 이것을 명확히 하는 것이 프레임워크의 핵심이다.

### 3단계: DAC 평가기준 매핑

OECD DAC 6대 평가기준(2019 개정)에 맞춰 프레임워크를 검증한다:

| DAC 기준 | 핵심 질문 | 프레임워크 연결 |
|---|---|---|
| **적절성(Relevance)** | 올바른 일을 하고 있는가? | 목표-니즈 정합성, 수원국 정책 연계 |
| **일관성(Coherence)** | 다른 개입과 조화되는가? | 타 사업/정책과의 시너지, 중복 방지 |
| **효과성(Effectiveness)** | 목표를 달성하고 있는가? | 성과(Outcome) 지표 달성도 |
| **효율성(Efficiency)** | 자원을 잘 활용하고 있는가? | 투입 대비 산출 비율, 예산 집행률 |
| **영향력(Impact)** | 어떤 차이를 만들었는가? | 영향(Impact) 수준 변화, 파급효과 |
| **지속가능성(Sustainability)** | 효과가 지속되는가? | 제도화, 역량이전, 재정적 자립 |

KOICA는 **범분야(Cross-cutting)**를 추가 기준으로 적용한다: 젠더, 환경, 인권, 장애포용.

상세 가이드 → `references/pcm-indicators-guide.md`

### 4단계: 성과 매트릭스 작성

성과 체인의 각 수준에 대해 지표, 기초선, 목표치, 데이터 수집 방법을 정의한다:

```
| 수준 | 성과 서술 | 지표 | 기초선 | 목표치 | 데이터 소스 | 수집 빈도 | 책임자 |
|---|---|---|---|---|---|---|---|
| 영향 | [상위목표] | [지표] | [값] | [값] | [소스] | 연 1회 | [담당] |
| 성과 | [사업목적] | [지표] | [값] | [값] | [소스] | 반기 | [담당] |
| 산출 1 | [산출물1] | [지표] | [값] | [값] | [소스] | 분기 | [담당] |
| 산출 2 | [산출물2] | [지표] | [값] | [값] | [소스] | 분기 | [담당] |
```

### 5단계: 데이터 수집 계획

각 지표에 대해:
- **수집 방법**: 설문조사, 인터뷰, 관찰, 행정 데이터, 시스템 로그
- **수집 도구**: 설문지, 체크리스트, 인터뷰 가이드, 데이터 추출 쿼리
- **수집 주기**: 월/분기/반기/연
- **표본 설계**: 전수조사 vs 표본조사, 표본 크기, 표본 추출 방법
- **책임자**: 누가 데이터를 수집하고 관리하는가
- **품질 관리**: 데이터 검증 절차 (data-quality-check 스킬 연계)

### 6단계: M&E 일정 통합

워크플랜과 M&E 일정을 통합한다:

```
| M&E 활동 | 시기 | 담당 | 비고 |
|---|---|---|---|
| 베이스라인 조사 | M1~M3 | M&E 전문가 | 착수보고서에 포함 |
| 분기 모니터링 | 매 분기말 | PM + M&E | 진도보고서 연동 |
| 중간평가 | 사업 중간 | 외부 평가팀 | 별도 TOR |
| 종료평가 | 사업 종료 전 | 외부 평가팀 | DAC 기준 적용 |
| 엔드라인 조사 | 사업 종료 전 | M&E 전문가 | 베이스라인 대비 |
```

## 출력 형식

```
# 성과 프레임워크: [사업명]

## 1. 변화이론 (Theory of Change)
[서술형 + 시각적 다이어그램]

## 2. 성과 체인
[투입 → 활동 → 산출 → 성과 → 영향]

## 3. 성과 매트릭스
[수준별 지표, 기초선, 목표치, 데이터 소스 표]

## 4. DAC 기준 매핑
[6대 기준별 평가 질문 및 지표 연결]

## 5. 데이터 수집 계획
[지표별 수집 방법, 도구, 주기, 책임]

## 6. M&E 일정
[워크플랜 연동 일정표]
```
