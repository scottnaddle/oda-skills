# 데이터 품질 점검

M&E 데이터의 품질을 체계적으로 평가하여, 신뢰할 수 있는 성과 보고와 의사결정의 기반을 확보한다.

## 왜 데이터 품질이 중요한가

"Garbage in, garbage out." 아무리 정교한 분석을 해도 데이터가 부정확하면 결론도 부정확하다. ODA 사업에서 데이터 품질 문제는 성과 보고의 신뢰성을 훼손하고, 잘못된 의사결정을 유발한다. 발주처(특히 ADB)는 데이터 품질 관리를 점점 더 중시하고 있다.

## 데이터 품질 5대 기준 (DQA Framework)

### 1. 완전성 (Completeness)
- 수집해야 할 데이터가 모두 수집되었는가?
- 결측값(Missing Data) 비율은 허용 범위 내인가?
- 모든 대상(지역, 집단, 기간)의 데이터가 포함되어 있는가?

**점검 방법:**
- 전체 데이터 건수 vs 예상 건수 비교
- 결측률 계산 (변수별, 응답자별)
- 기준: 결측률 10% 미만이 일반적 허용 범위

### 2. 정확성 (Accuracy)
- 데이터가 실제 값을 정확히 반영하는가?
- 입력 오류, 코딩 오류가 없는가?
- 이상값(Outlier)이 실제값인가 오류인가?

**점검 방법:**
- 범위 검증 (나이가 200, 점수가 -50 같은 불가능한 값)
- 논리 검증 (여성인데 군 경력 20년 같은 모순)
- 이중 입력(Double Entry) 비교
- 원본 문서 대조 (샘플 5~10%)

### 3. 일관성 (Consistency)
- 동일 대상의 데이터가 시점 간, 소스 간 일치하는가?
- 관련 변수 간 논리적 일관성이 있는가?

**점검 방법:**
- 시계열 일관성: 월별/분기별 데이터의 급격한 변동 확인
- 교차 검증: 서로 다른 출처의 동일 지표 비교
- 내적 일관성: 관련 문항 간 응답 패턴 확인

### 4. 적시성 (Timeliness)
- 데이터가 필요한 시점에 수집되고 보고되는가?
- 데이터 수집과 활용 사이의 시간차는 적정한가?

**점검 방법:**
- 수집 일정 준수율
- 데이터 보고 지연 현황
- 실시간 데이터 vs 지연 데이터 비율

### 5. 신뢰성 (Reliability)
- 동일 조건에서 반복 측정하면 같은 결과가 나오는가?
- 측정 도구(설문지, 체크리스트)가 표준화되어 있는가?
- 조사원 간 측정 편차가 없는가?

**점검 방법:**
- 조사원 간 신뢰도(Inter-rater Reliability)
- 재검사 신뢰도(Test-retest Reliability)
- 내적 일관성(Cronbach's alpha) — 설문 도구
