# /bid-analyze

RFP를 받았을 때 수주 전략 수립에 필요한 전체 분석을 순서대로 실행하는 워크플로우.

## 실행 흐름

### Step 1: RFP 분석 (analyze-rfp)

먼저 RFP/TOR 문서를 체계적으로 분석한다.

- 사용자가 파일을 첨부했으면 파일을 읽고 분석
- 텍스트로 공고 내용을 붙여넣었으면 해당 내용 분석
- 사업명만 제공했으면 알려진 정보 기반으로 분석 + 추가 정보 요청

**이 단계의 핵심 산출물:**
- 사업 개요 정리
- 평가기준 배점표 구조화
- 자격요건 체크리스트
- 투입인력 요구사항
- 리스크 초기 식별

분석이 완료되면 사용자에게 결과를 제시하고, 다음 단계로 넘어갈지 확인한다.

### Step 2: 경쟁사 매핑 (competitor-mapping)

RFP 분석 결과를 기반으로 예상 경쟁사를 분석한다.

- 사용자에게 알고 있는 경쟁사 정보를 물어본다
- 해당 분야/국가의 주요 플레이어를 식별한다
- 경쟁 매트릭스를 작성한다

**이 단계의 핵심 산출물:**
- 예상 경쟁사 3~5개사 프로파일
- 경쟁 매트릭스 (UBION vs 경쟁사)
- UBION의 상대적 포지션

### Step 3: 컨소시엄 전략 (consortium-strategy)

경쟁 분석 결과를 바탕으로 최적의 컨소시엄 구성안을 설계한다.

- 컨소시엄 필요성 진단 (단독 vs 컨소시엄)
- 필요 역할과 파트너 유형 정의
- 1~2개의 컨소시엄 구성안 제시

**이 단계의 핵심 산출물:**
- 컨소시엄 구성 권고안 (1안, 2안)
- 역할 배분 매트릭스
- 지분율 제안

### Step 4: 자가평가 (bid-scoring)

최종적으로 현재 상태에서의 수주 가능성을 종합 평가한다.

- RFP 배점표 기반 항목별 채점
- 강점/약점 종합
- Go/No-Go 권고

**이 단계의 핵심 산출물:**
- 항목별 자가 점수표
- 예상 총점 및 수주 전망
- Go/No-Go 판단과 핵심 개선 포인트

## 진행 원칙

- 각 단계가 완료될 때마다 결과를 제시하고, 사용자의 피드백을 반영한 후 다음 단계로 진행한다
- 사용자가 특정 단계를 건너뛰고 싶다면 유연하게 대응한다
- 정보가 부족한 부분은 가정(assumption)을 명시하고, 사용자에게 확인을 요청한다
- 최종 결과는 하나의 종합 보고서로 통합하여 제시한다

## 최종 산출물 구조

```
# 수주 분석 종합 보고서: [사업명]

## Executive Summary
[1페이지 요약: 사업 개요, 핵심 판단, Go/No-Go 권고]

## Part 1: RFP 분석
[analyze-rfp 결과]

## Part 2: 경쟁 환경 분석
[competitor-mapping 결과]

## Part 3: 컨소시엄 전략
[consortium-strategy 결과]

## Part 4: 수주 가능성 평가
[bid-scoring 결과]

## 다음 단계 액션 플랜
1. [즉시] ...
2. [1주 내] ...
3. [제안서 작성 시] ...
```

## 완료 후 제안

분석이 완료되면 자연스럽게 다음 워크플로우를 안내한다:

- Go 결정 시: "제안서 작성을 시작하시겠습니까? → `/draft-proposal`"
- 수원국 정보가 필요할 때: "수원국 현황을 더 조사하시겠습니까? → `/research-country`"
- 추가 공고를 검토할 때: "다른 공고도 검토하시겠습니까? → `/scan-procurement`"
