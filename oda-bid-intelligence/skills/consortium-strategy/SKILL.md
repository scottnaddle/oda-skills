# 컨소시엄 구성 전략

ODA 사업 입찰에서 최적의 컨소시엄(공동수급체)을 구성하기 위한 전략적 프레임워크.

## 왜 컨소시엄 전략이 필요한가

ODA 사업의 규모와 복잡성이 커지면서, 단독 입찰보다 컨소시엄 입찰이 일반적이다. 좋은 컨소시엄은 1+1=3을 만들지만, 잘못된 컨소시엄은 오히려 약점을 노출한다. 파트너 선정은 수주 확률에 직접적인 영향을 미치는 전략적 의사결정이다.

## 분석 절차

### 1단계: 컨소시엄 필요성 진단

먼저 컨소시엄이 정말 필요한지, 단독 입찰이 가능한지를 판단한다:

**컨소시엄이 필요한 경우**
- 자격요건(실적, 매출액)을 단독으로 충족하지 못할 때
- 사업 범위가 UBION의 핵심 역량을 넘어설 때 (예: 건축+ICT 융합)
- 현지 파트너 참여가 RFP에서 우대/필수일 때
- 특정 전문 분야의 인력을 자체 보유하지 못할 때

**단독 입찰이 유리한 경우**
- 자격요건을 자체 충족 가능할 때
- 이익 배분 이슈 없이 수익성을 극대화하고 싶을 때
- 의사결정 속도와 실행 효율을 우선할 때
- 사업 규모가 작아 분업이 비효율적일 때

### 2단계: 파트너 역할 정의

사업에 필요한 역할을 정의하고, 각 역할에 적합한 파트너 유형을 매핑한다:

**역할 유형**
- **주관사(Lead Firm)**: 전체 PM, 발주처 대응, 계약 관리
- **기술파트너**: 특정 전문 분야 담당 (예: TVET, 보건, 농업)
- **현지파트너**: 현지 네트워크, 로지스틱스, 현지 인력 제공
- **IT/시스템파트너**: 플랫폼 개발, 시스템 구축
- **학술파트너**: 연구, 커리큘럼 개발, 평가

**UBION의 자연스러운 포지션**
- 교육/ICT 사업의 주관사 또는 IT/시스템 파트너
- LMS/에듀테크 플랫폼 구축 역할
- 디지털 교육 콘텐츠 개발 역할
- AI 기반 교육 혁신 영역

### 3단계: 파트너 평가 매트릭스

후보 파트너를 다음 기준으로 평가한다:

| 평가 기준 | 가중치 | 설명 |
|---|---|---|
| 역량 보완성 | 30% | UBION의 약점을 얼마나 보완하는가 |
| 유사 실적 | 25% | 해당 분야/국가 실적이 입찰 평가에 기여하는가 |
| 핵심 인력 | 20% | 제안서에 투입 가능한 우수 전문가를 보유하는가 |
| 협업 이력 | 10% | 과거 협업 경험, 신뢰 수준 |
| 조직 안정성 | 10% | 재무 건전성, 사업 지속성 |
| 가격 합리성 | 5% | 인건비/간접비 수준 |

### 4단계: 지분율 및 역할 배분 설계

**지분율 결정 원칙**
- 주관사: 최소 50% (KOICA 기준), ADB는 Lead Firm이 주요 역할 수행 명시
- 지분율은 업무 범위(M/M)에 비례하는 것이 원칙
- 실적 인정을 위한 최소 지분율 확인 (보통 30% 이상이어야 실적으로 인정)

**역할 배분 매트릭스**

```
| 업무 영역 | 주관사(UBION) | 파트너 A | 파트너 B |
|---|---|---|---|
| 사업 총괄 관리 | ● | | |
| 교육과정 개발 | ● | ○ | |
| 플랫폼 구축 | ● | | |
| 현지 교육 실시 | ○ | ● | |
| M&E | ○ | | ● |
| 현지 로지스틱스 | | | ● |

●: 주 담당, ○: 보조/협력
```

### 5단계: 리스크 관리 방안

컨소시엄 운영에서 발생 가능한 리스크와 대응방안:

- **파트너 이탈 리스크**: MOU/계약에 중도 탈퇴 시 페널티 조항 포함
- **품질 리스크**: 성과물 품질 기준을 사전에 합의, 주관사 검수권 확보
- **소통 리스크**: 정기 회의체(주간/월간), 공유 프로젝트 관리 도구 활용
- **재무 리스크**: 지급 조건/일정 명확히 합의, 선급금 배분 규칙
- **지적재산권**: 공동 개발 결과물의 IP 귀속 사전 합의
